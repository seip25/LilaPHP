LilaPHP Framework

LilaPHP is a lightweight and modular PHP micro-framework focused on simplicity, flexibility, and security.

Build fast and independent web apps using clean routing, middleware, Twig templates, Dotenv configuration, auto-validation, and language management.

⭐ View LilaPHP on GitHub

Installation

  1. Clone the repository:
    2. git clone https://github.com/seip25/LilaPHP.git
  2. Navigate to app directory and install dependencies:
    3. cd app && composer install
  3. Create your environment file:
    4. cp .env.example .env
  4. Run locally:
    5. php -S localhost:8000

Routing

LilaPHP supports minimal, clean routing. You can define routes in any index.php at the root or within subdirectories. 

<?php
include_once "./app/index.php";

$app->get(callback: function($req, $res) use ($app): mixed {
    return $app->render("index");
});

$app->post(callback: function($req, $res) use ($app): mixed {
    return $app->jsonResponse(["success" => true]);
});

$app->run();

Language Switching

<?php
include_once "../app/index.php";

$app->get(callback: function ($req, $res) use ($app) {
    $newLang = $_GET["lang"] ?? $app->getLangDefault();
    $app->setSession(key: "lang", value: $newLang);
    $back = $_SERVER['HTTP_REFERER'] ?? '/';
    $app->redirect($back);
});

$app->run();

Validation Example

LilaPHP provides automatic validation through PHP 8 attributes using the #[Field] annotation. 

<?php
include_once "../app/index.php";

use Core\BaseModel;
use Core\Field; 

class LoginModel extends BaseModel
{
    #[Field(required: true, format: "email")]
    public string $email;

    #[Field(required: true, min_length: 6)]
    public string $password;
}

$lang = $app->getSession(key: "lang", default: "es");

$app->get(callback: fn($req, $res) => $app->render("login"));

$app->post(
    callback: function ($req, $res) use ($app, $lang) {
        return $app->jsonResponse(["success" => true, "session" => $lang]);
    },
    middlewares: [
        fn($req, $res) => new LoginModel(data: $req, lang: $lang),
    ],
    csrf: true
);

$app->run();

Sessions

Session handling is simple and secure. LilaPHP manages cookies with proper domain isolation and expiration. 

$app->setSession("user_id", 15);
$user = $app->getSession("user_id");

Translations

Store your translations in app/locales/ as JSON or PHP arrays. 

{
  "login": {
    "title": "Iniciar sesión",
    "email": "Correo electrónico"
  }
}

$app->t("login.title"); // Returns "Iniciar sesión"

SEO Optimization

LilaPHP automatically sets meta tags, Open Graph, and Twitter data from your configuration and template variables. It also supports dynamic images for link previews.

Performance & Minification

In production mode (APP_ENV=production), HTML, JS, and CSS output are automatically minified to improve performance.

Automatic Security

  • CSRF protection on POST requests.
  • Session and cookie isolation per domain.
  • Automatic input sanitization.
  • Strict Content-Security headers.
  • /app directory blocked by default.
